# C2PA_based_blocker

First iteration of a Chrome extension that scans page images, looks for C2PA/AI metadata signals, and replaces flagged images with a warning image.

## What this iteration includes

- Manifest V3 extension scaffold with content script, background worker, and options page.
- Image scanning for:
  - `<img>` tags
  - lazy image URL attributes (`data-src`, `data-original`, `data-lazy-src`)
  - simple CSS `background-image` usage
  - dynamically inserted elements via `MutationObserver`
- Heuristic detector that inspects image bytes for C2PA-like markers and AI-generation claims.
- Domain allowlist and strict-mode policy in options.
- Badge counter for number of replacements on the active tab.

## Important limitation (v0.1)

This version is **signal-based** and does **not perform cryptographic C2PA signature verification** yet. It is a practical MVP for flow and UX. A later iteration should use a full C2PA parser/verifier library and trust chain validation.

## Install locally

1. Clone this repository.
2. Open Chrome and navigate to `chrome://extensions`.
3. Enable **Developer mode**.
4. Click **Load unpacked** and select this project directory.
5. Open extension options to configure strict mode and allowlist.

## Behavior

- Default mode: replace only images flagged as AI-generated by detector logic.
- Strict mode: additionally replace images where C2PA-style metadata is found but verification is unavailable.
- Allowlist: comma/newline-separated domains where replacement is disabled.
